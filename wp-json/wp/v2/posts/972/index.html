AsyncRat: A Remote Administration Trojan (RAT)<\/p>\n

AsyncRat is a malicious Remote Administration Trojan (RAT) that has been actively used in cyber attacks since 2019. This malware is designed to compromise the security of a target system and allow remote attackers to gain unauthorized access and control. In this blog post, we will analyze the behavior and characteristics of AsyncRat to better understand how it operates and how to protect against it.<\/p>\n

How This AsyncRat Infection Works<\/p>\n

The AsyncRat is typically delivered to the victim through various methods, including phishing emails, social engineering tactics, and exploiting vulnerabilities in unpatched software. In some cases, the malware may also be delivered through drive-by downloads, where the victim unwittingly downloads the malware while visiting a compromised website. The exact method of delivery may vary depending on the attackers’ goals and the target’s specific vulnerabilities.<\/p>\n

In this instance it was delivered through malspam that contain a malicious Microsoft OneNote document within a zip archive named Invoice #099272663.zip. The malicious OneNote document (Invoice #099272663.one) contained a social engineering \u201cCLICK TO VIEW DOCUMENT\u201d image, that once clicked, it executes embedded VBScript.<\/p>\n

Figure 1: Embedded social engineering \u201c<\/a>CLICK TO VIEW DOCUMENT\u201d image in the OneNote document (Invoice #099272663.one)<\/h6>\n

The VBScript invokes PowerShell to download a benign decoy OneNote document and a malicious batch script from the following URLs:<\/p>\n

Download URL: hxxps:\/\/www.onenotegem[.]com\/uploads\/soft\/one-templates\/the_daily_schedule.one<\/p>\n

<\/a> Drop folder location: C:\\Users\\<Username>\\AppData\\Local\\Temp\\invoice.one (Classification: benign decoy OneNote document used to trick the victim into thinking that all is well, after clicking on the aforementioned embedded social engineering \u201cCLICK TO VIEW DOCUMENT\u201d image)<\/p>\n

Download URL: hxxps:\/\/transfer[.]sh\/get\/bjLvT5\/AsyncClientnownow.bat<\/p>\n

Drop folder location: C:\\Users\\<Username>\\AppData\\Local\\Temp\\system32.bat (Classification: ASyncRAT loader – malicious batch file)<\/p>\n

Figure 2: Embedded VBScript that when executed downloads a decoy OneNote document and a malicious batch script.<\/h6>\n

Spoilers!! The batch script (AsyncClientnownow.bat) that is downloaded to the system and executed, will ultimately drop and executed a ASyncRAT executable (payload.exe). The ASyncRAT executable is written in Microsoft .NET Framework. The batch script is classified as a ASyncRAT dropper.<\/p>\n

Next, I will briefly discuss how the heavily obfuscated batch script works. The first part of its code is an obfuscated executable file and the second part is an base64 encoded PowerShell script.<\/p>\n

Figure 3: Obfuscated batch script that contains an encrypted ASyncRAT binary.<\/h6>\n

Once the base64 encoded PowerShell script is run it is de-obfuscated during runtime. A review of this PowerShell shows that it is used to de-obfuscate the first part of code in the batch script utilizing base64 decode, AES Decryption (Key and IV), and Gunzip. The resulting de-obfuscated code turns out to be the aforementioned ASyncRAT executable file, which is then loaded into memory and executed.<\/p>\n

Figure 4: A portion of the de-obfuscated batch script that shows the PowerShell script.<\/h6>\n

During my manual analysis, I was able to decrypt the obfuscated executable file using CyberChef. As I mentioned earlier it is classified as ASyncRAT and written in Microsoft .NET Framework. I re-named this file payload.exe.<\/p>\n

Figure 5: De-obfuscating the first part of the batch script using CyberChef. The file header is MZ, revealing that it is a Windows executable file.<\/h6>\n

That ends Part 1 of this blog, which was just analyzing stage 1 of this infection. In Part 2 of this blog post I will review stage 2, which will cover the analysis of the ASyncRAT payload and dive into some of its functionalities.<\/p>\n
